# YouTube収録スライド生成システム — ユーザーマニュアル

## これは何？

長尺YouTube動画を **「一発撮り」** で収録するための、フルオート専用スライド生成ツールです。

台本を入力するだけで、以下のスライドが **連番PNG** として自動生成されます：

1. **ナビゲーションスライド** — 「今どこを話しているか」が一目でわかる全体マップ
2. **箇条書きスライド** — ポイントが1つずつ追加される「パラパラ漫画」式
3. **カンペ（台本）** — 画面下部に読み上げテキストを表示

収録時はスライドをクリックで送るだけ。台本を暗記する必要はありません。

## クイックスタート（3ステップ）

### 1. APIキーを入力

左サイドバーにAPIキーを入力します。

| プロバイダー | キーの形式 | 用途 |
|---|---|---|
| Google | `AIza...` で始まるキー | Geminiで箇条書き抽出 |
| OpenAI | `sk-...` で始まるキー | GPTで箇条書き抽出 |

キーはブラウザのlocalStorageに保存され、次回以降自動復元されます。

### 2. 台本を入力

テキストエリアに台本を書き、ステップの区切りに `[---STEP---]` を挿入します。

```
はじめに：今日のテーマ
今日はAIを使った動画制作の全体像をお話しします。
この動画を見れば、初心者でもAI動画を作れるようになります。

[---STEP---]

ステップ1：企画と台本づくり
まず最初にやるべきことは、企画と台本づくりです。
いきなりカメラの前に立つのではなく、
まず話す内容を整理しましょう。
台本があることで、話がブレず、テンポよく進められます。

[---STEP---]

ステップ2：スライドの準備
次に、スライドを準備します。
AIツールを使えば、台本からスライドを自動生成できます。

[---STEP---]

ステップ3：収録と編集
最後に、スライドを見ながらクリックで進めて一発撮りします。
```

**ルール：**
- 各ステップの **1行目** がそのステップのタイトルになります
- 2行目以降が台本テキスト（カンペとして表示されます）
- 「ステップ区切り挿入」ボタンでカーソル位置にタグを挿入できます

### 3. 生成ボタンを押す

「収録スライド生成」ボタンをクリック。あとは待つだけです。

## 生成される画像の順番

上の例（3ステップ）の場合、以下の順番で連番PNGが生成されます：

| # | 種類 | 内容 |
|---|---|---|
| 001 | ナビ | 全体の流れ（ステップ1がハイライト） |
| 002 | 箇条書き | ステップ1の箇条書き ●1つ目だけ表示 |
| 003 | 箇条書き | ステップ1の箇条書き ●1つ目 + ●2つ目 |
| 004 | 箇条書き | ステップ1の箇条書き ●1つ目 + ●2つ目 + ●3つ目 |
| 005 | ナビ | 全体の流れ（ステップ2がハイライト） |
| 006 | 箇条書き | ステップ2の箇条書き ●1つ目だけ表示 |
| 007 | 箇条書き | ステップ2の箇条書き ●1つ目 + ●2つ目 |
| ... | ... | ... |

収録時にこの順番でクリックするだけで、箇条書きがポンポンと出現し、ステップが切り替わるたびにナビゲーションが表示されます。

## 画面の見方

### ナビゲーションスライド

```
┌────────────────────────────────┐
│          全体の流れ              │
│                                 │
│  ① はじめに：今日のテーマ (グレー) │
│  ② ステップ1：企画  ← ハイライト  │
│  ③ ステップ2：スライド   (グレー)  │
│  ④ ステップ3：収録       (グレー)  │
│                                 │
└────────────────────────────────┘
```

- 現在のステップ → 赤いハイライト＋塗りの番号
- その他のステップ → グレーアウト

### 箇条書きスライド

```
┌────────────────────────────────┐
│ ■ ステップ1：企画と台本づくり     │  ← タイトル
│                                 │
│  ● まず話す内容を整理する       │  ← 既出の箇条書き（やや暗い）
│  ● 台本でテンポよく進行できる    │  ← 最新の箇条書き（明るい）
│                                 │
│─────────────────────────────── │
│ SCRIPT                          │
│ まず最初にやるべきことは、企画と   │  ← カンペ（読み上げテキスト）
│ 台本づくりです。                  │
└────────────────────────────────┘
```

- 最新の箇条書き → 赤いマーカー + 白テキスト（目立つ）
- 既出の箇条書き → 薄い赤マーカー + グレーテキスト
- 画面下部 → カンペ（実際に読み上げるテキスト）

## ダウンロード

### ZIP一括ダウンロード

生成完了後、画面上部に表示される「連番PNG (.zip)」ボタンをクリック。

- ファイル名を自由に設定可能（例：`ai-video-tutorial`）
- `ai-video-tutorial-001.png`, `ai-video-tutorial-002.png`, ... の連番で出力
- この順番のままOBS等で表示すれば収録完了

### 個別ダウンロード

各スライドにホバーして「保存」ボタンでPNG単体ダウンロード。

## 設定項目

| 項目 | 説明 |
|---|---|
| LLMプロバイダー | 箇条書き抽出に使うAI（Google Gemini / OpenAI GPT） |
| LLMモデル | Gemini 2.5 Flash（高速）/ Pro（高精度）/ GPT-4o 等 |
| スライドフォント | Noto Sans JP / M PLUS Rounded 1c 等から選択 |
| フォントウェイト | Regular〜Black（太さ） |

## 収録のコツ

1. **事前にZIPをダウンロード** して画像ビューアで開いておく
2. OBSやZoomの画面共有で画像ビューアを映す
3. 矢印キー（→）でスライドを送りながら話す
4. カンペを見ながら話すので、台本の暗記は不要
5. 箇条書きがポンポン出るので、視聴者のテンポ感も良好

## よくある質問

### Q: 箇条書きが的外れな内容になる
台本テキストが短すぎると、LLMが正確に要約できません。各ステップに最低3〜4行の説明文を書いてください。

### Q: カンペのテキストが途切れる
カンペエリアは最大4〜5行です。長い台本テキストは自動的に分割され、各箇条書きスライドに振り分けられます。

### Q: フォントが反映されない
Google Fontsの読み込みに少し時間がかかる場合があります。ページをリロードしてから再生成してください。

### Q: APIエラーが出る
- APIキーが正しいか確認してください
- Google APIの場合、Gemini APIが有効になっているか確認してください
- 無料枠の上限に達している場合があります

### Q: 停止ボタンを押したらどうなる？
生成済みのスライドはそのまま残ります。途中までのスライドをダウンロードすることも可能です。
